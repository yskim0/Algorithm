# 코드플러스 알고리즘 기초 - 2/2
## 620 - 트리


트리
- 사이클이 없는 연결 그래프
- 정점 : V
- 간선 : V-1
- 트리는 그래프이기 때문에, 그래프의 표현과 같은 방식으로 저장할 수 있다
- 완전 이진 트리의 경우에는 배열로 표현할 수 있다
- 이진 트리의 경우에는 구조체나 클래스 이용



문제) 트리의 순회
- 노드 방문 처리를 언제할 것이냐에 따라 DFS는 아래와 같이 3가지 출력 순서가 있다
- 프리오더
- 인오더
    - BST 삭제 구현할 때 인오더의 successor 구함. 인오더 순서를 사용함
    - 이진트리가 아니면 정의 안됨
- **포스트 오더**
    - 제일 많이 씀
